Take a look at this partial implementation of a system with support for byte-sized memory accesses. The attacker and the victim program do not share memory, but they do use the same direct-mapped cache with a size of 32 bytes and a block size of 1 byte. The victim program leaks the secret FLAG via a cache side channel. Your tasks is to determine the value of FLAG by performing memory accesses and running the victim program. Use your browser console to make sys.attackerAccess() and sys.victimProgram() function calls and capture the flag! For background information on caches, see here. 

const FLAG;

const MEMORY_SIZE = 128;
const VICTIM_MEMORY_START = 0;
const VICTIM_MEMORY_END = 64;
const ATTACKER_MEMORY_START = 64;
const ATTACKER_MEMORY_END = 128;

const CACHE_SIZE = 32;

class System {
  attackerAccess(address) {
    if (address < ATTACKER_MEMORY_START || address >= ATTACKER_MEMORY_END) {
      console.log("SEGMENTATION FAULT");
      return;
    }
    return this.#access(address);
  }

  victimProgram() {
    for (let i = 0; i < FLAG.length; i++) {
      this.#victimAccess(FLAG[i].charCodeAt() - 65);
    }
    console.log("VICTIM PROGRAM COMPLETE");
  }
}

sys = new System();
Hint 1: Begin by "priming" the cache, or filling the cache with known memory blocks, using sys.attackerAccess() calls. Notice that each initial access takes 3 seconds to retrieve the block from main memory, but each further access of the same block is much faster since it is stored in the cache. Hint 2: Call sys.victimProgram() to run the victim program, which replaces certain cache blocks based on the value of FLAG. Hint 3: "Probe" the cache by making sys.attackerAccess() calls and observing how long they take. If an access takes 3 seconds, the block must have been replaced by the victim program and needed to be retrieved again from main memory. Hint 4: Figure out which cache blocks are associated with which characters and unscramble the letters to find FLAG's value!
